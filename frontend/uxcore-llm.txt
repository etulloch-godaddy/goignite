## Box

The Box component is a foundational layout component that provides consistent spacing and alignment capabilities.

### Installation
```sh
npm i @ux/box
```

### Overview
Box serves as an organizational container component that helps maintain consistent spacing and alignment in layouts. It uses logical properties to handle different writing modes and provides multiple ways to align and space child elements.

### Key Features
- Flexible orientation (vertical/horizontal)
- Logical property support for RTL
- Built-in spacing system
- Child alignment controls
- Self-alignment capabilities
- Automatic wrapping option

### Props
| Prop | Type | Description |
| ---- | ---- | ----------- |
| `className` | `String` | Class for positioning |
| `orientation` | `'horizontal' \| 'vertical'` | How children appear within the component. Default is `vertical` to suggest block stacking behavior |
| `wrap` | `Boolean` | Should children wrap when room isn't available (works with `orientation='horizontal'`) |
| `blockAlignChildren` | `'start' \| 'center' \| 'end' \| 'spaceBetween'` | Sets the block-axis alignment of children |
| `inlineAlignChildren` | `'start' \| 'center' \| 'end'` | Sets the inline-axis alignment of children |
| `blockAlignSelf` | `'start' \| 'center' \| 'end'` | Sets the block-axis alignment of itself when child of another Box |
| `inlineAlignSelf` | `'start' \| 'center' \| 'end'` | Sets the inline-axis alignment of itself when child of another Box |
| `gap` | `'sm' \| 'md' \| 'lg' \| 'xl'` | Space between children |
| `inlinePadding` | `'sm' \| 'md' \| 'lg' \| 'xl'` | Padding in the inline axis |
| `blockPadding` | `'sm' \| 'md' \| 'lg' \| 'xl'` | Padding in the block axis |
| `displayType` | `String` | CSS display property, defaults to 'flex' |
| `stretch` | `Boolean` | Stretches available space (similar to flex: 1) |

### Usage Examples

```jsx
// Basic Box
import Box from '@ux/box';

function BasicExample() {
  return (
    <Box blockAlignChildren='center'>
      <p>Centered content</p>
      <Button text="Click me" />
    </Box>
  );
}

// Horizontal Layout with Gap
function HorizontalExample() {
  return (
    <Box 
      orientation='horizontal' 
      gap='md'
      wrap={true}>
      <Card>Item 1</Card>
      <Card>Item 2</Card>
      <Card>Item 3</Card>
    </Box>
  );
}

// Complex Alignment
function AlignmentExample() {
  return (
    <Box 
      blockAlignChildren='center'
      inlineAlignChildren='start'
      blockPadding='lg'
      inlinePadding='md'>
      <h2>Title</h2>
      <p>Content with padding</p>
    </Box>
  );
}
```

### Hook Usage
The `useBox` hook provides a way to apply Box functionality to other elements:

```jsx
import { useBox } from '@ux/box';

function CustomContainer() {
  const boxProps = useBox({ 
    blockAlignChildren: 'center',
    gap: 'md'
  });
  
  return (
    <div {...boxProps}>
      <p>Custom container content</p>
    </div>
  );
}
```

### Component Usage
For component-based usage, the package exports a `box` object with HTML tag references:

```jsx
import { box } from '@ux/box';

function SectionExample() {
  return (
    <box.section blockAlignChildren='center'>
      <h2>Section Title</h2>
      <p>Section content</p>
    </box.section>
  );
}
```

## Form Components

### Text Entry

### Installation
```sh
npm i @ux/text-entry
```

### Overview
The component is often meant to be used in conjunction with `<FieldFrame/>` as a replacement for a native HTML `<input/>` when building custom form controls. The `<TextEntry/>` component is designed to be _visually invisible_ without the `<FieldFrame/>` component, which provides a border (or other affordance) to signify a field for text input.

### Props
| Prop | Type | Description |
| ---- | ---- | ----------- |
| `className` | `String` | Sets a classname on the root component for use in layout placement. |
| `center` | `Boolean` | Centers the value within the input. |
| `disabled` | `Boolean` | Sets `disabled` on the `<input/>`. |
| `id` | `String` | **Required** Sets an `id` attribute on the `<input/>`, will also set on `for` attribute on label if provided. |
| `invalid` | `Boolean` | Set style and properties to reflect invalid state. |
| `label` | `String` `React.Component` | Creates a floating label over the input. When a component is given, it is cloned with additional classnames. | 
| `multiline` | `Boolean` | Replaces `<input>` with `<textarea>`. |
| `placeholder` | `String` | Placeholder text for the `<input/>`, if this and `label` are both provided, the `label` will not be shown. |
| `readOnly` | `Boolean` | Sets `readonly` on the `<input/>`. |
| `resize` | `String|Boolean` | For `multiline` use, allows the box to be resized. Allows for any CSS `resize` prop or `Boolean`. `true = 'both'` `false = 'none'` |
| `required` | `String` | Changes the style of the input to reflect a required state. This does _not_ set the `required` property on the `<input/>` but `aria-required` instead. |
| `size`| `Number` | Sizes the input. When using `multiline` this affects the vertical size. However, this will not work with floating labels; the label will determine the size. |
| `suggestion` | `String` | This is text that can allow for a tab to autocomplete functionality to be created. An example in provided in the `/examples` directory. The text here should start with the text currently in the `<input/>` value. |
| `type` | `String` | The type of `<input/>`, this only allows for text types (`email`, `url`, etc) |
| `value` | `String` | The value of the `<input/>`. |
| `visualSize` | `('sm' \| 'md')` | Sets component size, passed via ref from the convenience component |

### onChange Event
The `onChange` signature for this component is:
```js
onChange(theUpdatedValueExpected, { event: theOriginalEventFired });
```

### Usage Example
```jsx
import FieldFrame from '@ux/field-frame';
import TextEntry from '@ux/text-entry';

export default function MyInput() {
  return (
    <>
      <label htmlFor='my-input'>Username</label>
      <FieldFrame>
        <TextEntry 
          id='my-input'
          type="text"
          label="Username"
          required
        />
      </FieldFrame>
    </>
  );
}
```

### Password

### Installation
```sh
npm i @ux/password
```

### Overview
The Password component builds on TextEntry and FieldFrame to provide a secure password input with optional visibility toggle and validation features.

### Key Features
- Built on TextEntry foundation
- Optional visibility toggle
- Help text support
- Error message display
- Flyout content support

### Props
Inherits props from `@ux/text-entry` and `@ux/field-frame`, with these additions:

| Prop | Type | Description |
| ---- | ---- | ----------- |
| `button` | `Element` | Optional `@ux/button` above input |
| `children` | `Node` | Flyout content |
| `className` | `String` | Root component class |
| `helpMessage` | `String` | Helper/hint text |
| `errorMessage` | `String` | Error message |
| `open` | `Boolean` | Flyout visibility control |

### Event Handling
The `onChange` callback provides a consistent signature:
```js
onChange(value: string, { event: originalEvent });
```

### Usage Examples
```jsx
// Basic Password Input
const BasicPassword = () => (
  <Password
    id="login-password"
    label="Password"
    required={true}
  />
);

// With Validation Flyout
const ValidatedPassword = () => {
  const [value, setValue] = useState('');
  
  return (
    <Password
      id="signup-password"
      label="Create Password"
      value={value}
      onChange={(val) => setValue(val)}
      helpMessage="Must be at least 8 characters"
    >
      <Criteria invalid={value.length < 8}>
        Minimum 8 characters
      </Criteria>
    </Password>
  );
};
```

### Breaking Changes in v2102
- Removed dependency on `@ux/form-element`
- Converted to functional component with ref forwarding
- Removed built-in validation
- Developer-managed password visibility
- Required `id` prop for accessibility
- Future integration with `@ux/criteria` planned

## Flyout

### Installation
```sh
npm i @ux/flyout
```

### Overview
A base component for all items that are visually anchored to another element, expected to appear on top of the UI. Examples include dropdowns and tooltips.

### Props
| Prop | Type | Description |
| ---- | ---- | ----------- |
| `anchorRef` | `React ref` | Reference to the DOM node to attach the flyout visually |
| `arrow` | `Boolean` | Display an arrow connecting to the anchor |
| `placement` | `String` | Determines flyout placement (see Placement section) |
| `portalElem` | `Element` | Element to attach the flyout to, helps prevent overflow clipping |
| `strategy` | `String` | Positioning strategy: `absolute` or `fixed` |
| `role` | `String` | ARIA role applied to the flyout |
| `onClose` | `Function` | Callback fired when event occurs outside flyout/anchor |
| `onPlacement` | `Function` | Callback fired on placement change |
| `middleware` | `Array` | Array of floating-UI middleware configurations |

### Placement Options
Base placements:
- `auto`
- `top`
- `right`
- `bottom`
- `left`

Variations (`-start` or `-end` can be appended):
- `top-end` (on top, right aligned)
- `right-start` (on right, top aligned)
- `bottom` (on bottom, centered)
- `auto-end` (automatic side, alignment based on space)

### Middleware
The component exposes wrapper functions for common floating-UI middleware:

- `@ux/flyout/middleware/flip` - Allows flyout to flip when overflowing
- `@ux/flyout/middleware/stretch` - Stretches flyout to nearest relative ancestor
- `@ux/flyout/middleware/resize` - Dynamically resizes flyout to fit viewport
- `@ux/flyout/middleware/offset` - Adds spacing between flyout and anchor

### Usage Example
```jsx
import Flyout from '@ux/flyout';

export default function MyFlyout() {
  const [show, setShow] = useState(false);
  const anchorRef = useRef(null);

  return (
    <>
      <button
        ref={anchorRef}
        onClick={() => setShow(!show)}
        aria-labelledby="my-flyout">
        Toggle Flyout
      </button>

      {show && (
        <Flyout 
          anchorRef={anchorRef}
          id="my-flyout"
          placement="bottom"
          arrow={true}
          onClose={() => setShow(false)}>
          <p>Flyout content here</p>
        </Flyout>
      )}
    </>
  );
}
```

## Button

A comprehensive set of styled buttons following UXCore 2.0 design patterns.

### Installation
```sh
npm i @ux/button
```

### Key Features
- Seven design variants (critical, inline, primary, secondary, control, option)
- Icon support
- Internationalization support
- Toggle and select button modes
- Accessibility features built-in

### Design Variants
- `critical` - Red button for destructive actions
- `inline` - Actionable target within text (like links)
- `primary` - Main action button
- `secondary` - Lesser priority actions
- `control` - Used within form controls
- `option` - Items in selectable lists

### Best Practices
- Use `text` prop instead of children for content
- Use `icon` prop for adding icons
- Use `as` prop for special button types (select, toggle)
- Apply layout styles through className, not appearance styles

## Thumb

An interactive draggable element used in slider and range components.

### Installation
```sh
npm i @ux/thumb
```

### Key Features
- Drag interaction support
- Percentage-based positioning
- RTL support
- Custom precision
- Touch device support
- Accessibility considerations

### Props
| Prop | Type | Description |
| ---- | ---- | ----------- |
| `anchorRef` | `Element` | Reference element for movement |
| `onChange` | `Function` | Position change handler |
| `x` | `Number` | Horizontal position (0-100) |
| `y` | `Number` | Vertical position (0-100) |
| `rtl` | `Boolean` | Right-to-left support |
| `precision` | `Number` | Decimal place precision |
| `className` | `String` | Optional custom class |

### Usage Example
```jsx
import Thumb from '@ux/thumb';

function SliderExample() {
  const ref = React.useRef();
  const [position, setPosition] = React.useState(0);
  
  return (
    <div ref={ref}>
      <Thumb
        anchorRef={ref}
        x={position}
        y={50}
        onChange={(x, y) => setPosition(x)}
      />
    </div>
  );
}
```

### Accessibility
Note that the Thumb component is typically used as part of larger interactive components (like sliders or toggles) and relies on the parent component to implement keyboard interactions and ARIA attributes appropriate for the specific use case.
